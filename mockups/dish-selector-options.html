Mobile Dish Selector - 3 Alternative Approaches

Option 1: Bottom Sheet

A drawer that slides up from the bottom of the screen, similar to native iOS bottom sheets. Uses vertical space efficiently and feels natural on mobile.

Pros

  • Native mobile UX pattern
  • Easy thumb access
  • Handles long lists well
  • Clear entry/exit animation

Cons

  • Partial screen coverage
  • May need custom component
  • Different from desktop

Option 2: Full Screen Modal

A dedicated full-screen view for dish selection. Maximum space for search and browsing. Slides in from the right like a navigation push.

Pros

  • Maximum screen space
  • No overlap issues
  • Clear focus on task
  • Easy to implement

Cons

  • Loses context of meal list
  • Extra navigation step
  • Feels heavier

Option 3: Adaptive Popover (Recommended)

Keeps the current popover approach but makes it responsive. Full width on mobile, fixed width on desktop. Maintains consistency across devices with minimal code changes.

Pros

  • Minimal code changes
  • Desktop unchanged
  • Quick context switching
  • Same component reused

Cons

  • Less discoverable
  • May feel cramped
  • Overlay might obscure

My Recommendation: Option 3 (Adaptive Popover)

Why: It requires minimal code changes, maintains desktop behavior exactly as-is, and provides a functional mobile experience. The implementation would involve:

  • Adding responsive width classes to the popover (full width on mobile, 450px on desktop)
  • Adjusting max-height for better mobile fit
  • No new components needed
  • Works within existing shadcn-svelte/bits-ui architecture

Note: Option 1 (Bottom Sheet) would provide the best mobile UX but requires more development effort. Choose this if you want a premium mobile experience.
