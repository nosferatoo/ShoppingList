Meal Planner Mobile Interface - Design Options

Comparing three different approaches for the mobile dish selection interface. Desktop layout remains unchanged. Scroll down to see mockups and pros/cons.

Option 1

Compact Horizontal

Optimized version of current layout with smaller padding, fonts, and narrower date column. Maintains horizontal layout but fits mobile screens better.

Pros

  • Minimal changes to current design
  • Consistent with desktop (just scaled)
  • Shows more items at once
  • Quick to scan dates

Cons

  • Still feels cramped on small screens
  • Dish names may truncate heavily
  • Small touch targets (28px)
  • Popover still needs to be narrowed

Option 2

Vertical Stacked

Date and dish selector stacked vertically within each card. More breathing room, easier to read, better touch targets.

Pros

  • Larger touch targets (better UX)
  • Dish names don't truncate
  • More breathing room, cleaner
  • Easier to scan and interact

Cons

  • Takes more vertical space
  • See fewer items at once
  • More scrolling required
  • Different from desktop layout

Option 3

Minimal Card + Bottom Sheet

Clean card layout showing date and current dish. Tap opens a bottom sheet for dish selection (common mobile pattern).

Pros

  • Most native mobile experience
  • Cleanest, most readable layout
  • Full-width selector in sheet
  • No popover width issues

Cons

  • Requires bottom sheet component
  • Extra tap to open selector
  • Most different from desktop
  • More development effort

Implementation Notes:
  • Option 1: Quickest to implement - mainly CSS adjustments with responsive classes
  • Option 2: Moderate effort - restructure layout with media queries, good balance
  • Option 3: Most effort - need bottom sheet component (could use Dialog on mobile)

All options maintain the existing desktop layout unchanged. The popover width would be adjusted to calc(100vw - 2rem) on mobile for Options 1 & 2.
