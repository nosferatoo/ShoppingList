SHARED COMPONENT

Key Points:

  • Single Source of Truth: All meal planning logic lives in MealPlannerContent.svelte
  • Identical Experience: Both desktop and mobile use the exact same component with the same UI, features, and behavior
  • Desktop Context: MealPlannerContent is wrapped in a Dialog component (modal overlay)
  • Mobile Context: MealPlannerContent is rendered directly in the main content area when viewMode === 'meals'
  • Zero Duplication: Calendar view, dish selection, add/delete, confirm menu - all shared
  • Easy Maintenance: Fix a bug or add a feature once, it works everywhere
  • Responsive: The content component can adapt its layout based on available space if needed
  • State Management: All meal planning state (menus, dishes, loading) is managed within MealPlannerContent
